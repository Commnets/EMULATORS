Detailed Description

Representing a Integer, with and without sign.
It might by said that this class represents actually CPU's ALU (Arithmetic Logic Unit) as well.
A number is represented by several UBytes. Those UBytes can be given either in big-endian format or low-endian one.
That number can be represented in many different formats.
The _BINARY one is the default and the most commonly used.
The type of format used in many computers at 80's was the packaged BCD.
The class is prepared to support additional formats further than those two ones. It can be extended somehow. See later.
The negative numbers are always kept and "understood" using complement 2.
The basic operation in an UInt is "adding". The rest are always refer to this one.
