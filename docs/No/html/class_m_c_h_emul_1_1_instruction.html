Constructor.

Parameters
c: The internal code of the instruction.
mp: The number of memory positions (in number of ubytes) occupied by the instruction (and params).
cc: The number of clock cyles the instruction uses to be executed (usually).
Some ocassions, some instructions under certain circunstances can take longer than expected.
See also
for so: _additionalClockCycles (method and variable).
Parameters
t: The template of the instruction.
The way it is defined is key for it to be read and understood by the parsers.
The parameters should be within [] with two additional data:
The type of the parameter, and the number of bytes that parameter occupies.
Regarding the type: # means number, $ means address and & means relative jump (
See also
type of parameter).
